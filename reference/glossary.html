Glossary · 3D-rotations

所有课程共用的词汇。把整个项目压成两层：旋钮（knobs） × 映射曲线（curves）。

① 舞台 — The 3D Stage

perspective（透视）
设在父元素上的"相机到屏幕的距离"，单位 px。值越小 = 相机越近 = 3D 透视越夸张。 没有它，rotationX/Y 看起来是平的（没有近大远小）。本项目设在 .gallery__item-wrap 上：perspective: 900px。
transform-style: preserve-3d
设在被转的元素上，告诉浏览器"在真正的 3D 空间里渲染"，而不是先压平再贴上去。本项目设在 .gallery__item 上。

② 旋钮 — Knobs（每个 variation 转的东西）

旋钮轴 / 含义生活类比
rotationX绕水平轴翻（pitch / 俯仰）卡片向你/远离你"点头"地倒下
rotationY绕竖直轴转（yaw / 偏航）像一扇门 / 转盘左右转
rotationZ绕指向你的轴转（roll / 滚转）时钟指针在平面内打转
z沿"指向你"的轴平移（深度）z 负 = 推远变小；z 正 = 拉近变大（需 perspective 才看得见）
scaleX / scaleY横向 / 纵向缩放压扁拉长（squash & stretch）
filterblur / saturate / brightness模糊、饱和度、亮度——做"虚化/显影"

③ 引擎 — The Scroll Engine

ScrollTrigger + scrub: true
把动画进度直接绑到滚动条：你滚多少，动画走多少（可往回）。
self.progress
元素穿过触发区间时的进度：0=刚进入，0.5=正中，1=离开。 这是唯一的输入信号。所有花样都是把这个 0→1 映射成数值。
start / end
触发区间。本项目：start:'top bottom+=20%'，end:'bottom top-=20%'（元素从屏幕下方一点进、到上方一点出）。

④ 映射曲线 — Curves（"为什么不同"的核心）

把 progress (0→1) 变成实际数值的函数。同样的旋钮，换条曲线，感觉天差地别。
表达式形状 / 作用
interpolate(a, b, p)从 a 线性走到 b（p=0→a, p=1→b）。最朴素的"从…到…"。
Math.sin(p * π)两端=0、正中=1 的钟形。常用于"正中时最深/最亮"，两端归零。
Math.cos(p * π)p=0→1、正中→0、p=1→-1。常用于"两端强、正中过零"。
pow(sin(pπ), n)n 越大，钟形越尖：两端更平、正中更突然（sin⁴/sin⁸ = 猛地穿透）。
sign(x)·|x|^k保留正负号但压/拉幅度，做"带缓动的全程摆动"。
holdAtMiddle(p, hold)把进度在 0.5 处停住一段，让卡片在正中"定格展示"。（Variation 5 自定义）

⑤ 布局与平滑

positionGalleryItems()
用 Math.sin(i * 步长) * amplitude 给每个 wrap 一个左右偏移，形成蛇形排布。步长/amplitude 不同 → 排布疏密不同。
Lenis
平滑滚动库。和 GSAP ticker 接通，让 scrub 丝滑。Variation 4 还从它读 velocity（滚动速度）。
词汇表是"压缩过的本质"，每节课都遵守这里的命名。 主来源：MDN · Using CSS transforms · GSAP · ScrollTrigger
