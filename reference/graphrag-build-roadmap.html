GraphRAG From Scratch

从零重写一条最小可用的 GraphRAG 管线 · 分阶段 · 分 milestone

Mission. 不靠 microsoft/graphrag 这个包，自己用 Python 实现一条精简版的 indexing + query 管线，能 index 一个语料、能 query 出 flat 向量 RAG 答不出的「全局问题」，并能打包部署、在新机器上跑起来。 路线图按「指导一个 junior」的口吻写：每个 milestone 都有一个能跑起来、能验收的产物。

Part A · 这个项目是什么

GraphRAG 不是一个模型、也不是聊天机器人，而是一条数据管线： 把一堆非结构化文本，加工成一张知识图谱 + 多层社区摘要，让 LLM 能回答 普通向量 RAG 答不了的「纵观全局」问题（如「这批文档的核心主题有哪些？」）。

它分两半。Indexing（建索引）把文本变成可查的知识；Query（查询）在这套知识上回答问题。

Documents→ Chunk → TextUnits→ LLM 抽 Entities + Relationships→ 合并/摘要→ Leiden 社区检测→ 社区报告 (LLM)→ Embeddings → 向量库

这正是官方 6 个 phase（见 graphrag/docs/index/default_dataflow.md）。产物是一组 Parquet 表 +一个向量库，统称 Knowledge Model。

一句话抓住它的灵魂。 普通 RAG 只会「检索几个相关 chunk 再让 LLM 拼答案」——所以问「全局主题」时， 它根本没有哪几个 chunk 能代表全局，必然答不好。

GraphRAG 的杀手锏是 community reports：先把整个语料聚成多层社区、 每层都让 LLM 写一份摘要。问全局问题时，用 map-reduce 扫这些摘要 （每份给个局部答案 → 汇总成最终答案）。这就是它能答全局、而你要重点重写的部分。

Part B · 路线图（8 个 milestone）

原则：纵切，不横切。不要先把所有数据结构写完再写 LLM。每个 milestone 都端到端、能跑、能眼见为实。先在 M0 造一个会失败的「稻草人」，最后在 M5 亲手把它打败—— 这个反差就是你对 GraphRAG 的真正理解。

M0 · 稻草人：朴素向量 RAG Foundations

目标
先建立「为什么需要 graph」的痛感，并补齐 RAG 基本功（你这块是浅碰）。
你要造
最朴素的 RAG：chunk → embed → 余弦检索 top-k → 塞进 prompt → LLM 答。语料用小文本（如 A Christmas Carol，官方 quickstart 同款）。
学到
chunking、embedding、向量检索、prompt 拼接、LLM API 调用、token 计费直觉。
坑
chunk 太大/太小；忘了存 source 引用；把 API key 写进代码。
验收：一个 baseline.py：问 Scrooge 是谁（局部问题）答得不错； 问「全书主题有哪些」（全局问题）明显答得空泛/片面。把这个失败截图存下来。

M1 · 数据模型 + 管线骨架 Foundations

目标
定义 Knowledge Model，搭好「文档进 → 表出」的骨架。
你要造
用 pydantic/dataclass 定义 Document / TextUnit / Entity / Relationship / Community / CommunityReport。实现 chunking 生成带 id 与 provenance（指回源 chunk）的 TextUnits。一个极简 storage 层（先就写 Parquet/JSON）。CLI 入口 myrag index。
学到
数据建模、provenance（可溯源）、把「存储」抽象成接口（为后面 factory 埋伏笔）。
坑
id 不稳定导致重跑结果错乱；存储和业务逻辑耦死，后面换不了。
验收：myrag index 跑完，产出 text_units.parquet，每行带 id + 源文档引用。

M2 · 图抽取（LLM 核心） 核心

目标
把文本变成图：从每个 TextUnit 抽实体和关系。
你要造
抽取 prompt（直接研读 graphrag/.../prompts 里的 entity-extraction prompt 再自己写）；解析 LLM 输出为结构化 entities/relationships；合并去重（同 title+type 合一，描述并成数组）；摘要（让 LLM 把描述数组压成一句）；加一层 LLM cache（同输入直接返缓存）。
学到
结构化抽取 prompt、解析非确定性输出、实体消歧/合并、用 cache 让开发循环又快又便宜、idempotent 设计。
坑
LLM 输出格式漂移（要容错解析/重试）；不做 cache 会烧钱烧到肉疼；合并规则太粗导致同名异义混在一起。
验收：产出 entities.parquet + relationships.parquet；用 networkx 画出来——你已经有一张知识图谱了。

M3 · 社区 + 报告（杀手锏） 核心

目标
给图聚类成多层社区，并为每个社区生成报告——这是 GraphRAG 区别于一切普通 RAG 的地方。
你要造
用 networkx 建图 → Hierarchical Leiden 社区检测（库：graspologic（官方用）或 leidenalg+igraph），拿到多层社区归属；自底向上为每个社区用 LLM 写 community report（含概述 + 关键实体/关系）。
学到
图聚类直觉、层级 community 的意义（高层=全局视角、低层=局部簇）、把图结构「翻译」成自然语言摘要。
坑
把社区算法当黑箱不理解层级，后面 global search 不知道选哪层；大社区报告超 context，要先裁剪/分批。
验收：产出 communities.parquet + community_reports.parquet。索引数据已完整。

M4 · Embeddings + 向量库 核心

目标
给需要语义检索的产物建向量，落到向量库。
你要造
embed text units / entity descriptions / community report 文本；写入一个简单向量库（先内存 numpy 余弦，或 lancedb）；封装一个 search(text)→ids。
学到
该 embed 什么、向量库接口设计（又一个 factory 伏笔）、批量 embedding 的成本/限速。
坑
embedding 模型和检索时不一致；忘了存 id 映射。
验收：向量持久化；search("Scrooge") 能返回相关实体/文本的 id。

M5 · 查询引擎：Local + Global 核心

目标
在知识上回答问题，并亲手打败 M0 的稻草人。
你要造
Local search：query→找相关实体→收集邻域（实体/关系/源 text units/相关社区报告）→拼 context→LLM 答（擅长具体问题）。
Global search：对 community reports 做 map-reduce——map：每份报告给一个局部答案+打分；reduce：合并高分局部答案成终答（擅长全局问题）。
学到
context 预算管理、map-reduce over LLM、两种检索路径各自适配什么问题——这正是「该用什么」的迁移能力。
坑
context 塞爆；global 不分层、把所有报告一锅端；不给来源导致没法溯源。
验收：myrag query --method global "全书主题有哪些" 给出比 M0 明显更全面的答案。闭环完成 🎯

M6 · 产品化：config / CLI / 部署 Ship

目标
从「能在我电脑跑」到「能部署、新机器一键跑」。
你要造
settings.yaml + .env（key 不进代码）；myrag init；用 factory/provider 模式让 LLM/向量库/存储可替换（这是官方真正的扩展性设计）；成本闸（cache、并发限速、便宜模型、小数据起步）；打包 pip install . + Dockerfile；可选一个薄 FastAPI 把 query 暴露成服务。
学到
配置/密钥管理、依赖注入式可插拔架构、容器化、把脚本变服务。
坑
硬编码 provider 导致换模型要改源码；Docker 里忘了传 env；一上来就拿大语料把钱烧光。
验收：在一台干净机器上 docker run → init → index → query 全程跑通。「能部署可运行」达成。

M7 · 进阶（stretch）：评估 / 调优 / 增量 Ship

目标
从「能跑」到「跑得好、跑得省」。
你要造
prompt tuning（auto + manual）；答案质量评估（对比 M0 baseline）；增量索引（新文档进来不全量重跑）；可选 claims/covariates 抽取、DRIFT search。
学到
评测 RAG 质量、prompt 对结果的杠杆、增量更新的工程难点。
这一阶段按需取用，不是 MVP 必需。

Part C · 随用随补的技能（你 RAG/LLM 浅碰 → 这些缺口按需补）

LLM API & 结构化输出chat/embeddings 调用、JSON/schema 约束输出、重试与限速、token 计费。→ 在 M0/M2 现学现用。
RAG 基本功chunking 策略、embedding、向量检索、context 拼接。→ M0 一次性补齐。
图 & 社区检测networkx 基本操作、Leiden/Louvain 直觉、层级聚类。→ M3 现学。
数据工程pydantic 建模、Parquet（pandas/pyarrow）、可溯源 id 设计。→ M1。
工程化CLI（typer/argparse）、yaml/.env 配置、factory 模式、Docker、(可选) FastAPI。→ M6。
