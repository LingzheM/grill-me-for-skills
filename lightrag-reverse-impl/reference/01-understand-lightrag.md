# 01. 读懂原项目 → 项目地图

本文件两部分：上半部分是**通用的快速读码套路（任何项目都适用）**，下半部分是**针对 LightRAG 已经抽取好的“项目地图”**（标准答案，供 junior 自己读完后对照）

---

## A. 快速读码套路： 1-2 小时建立全局认知

读一个陌生的大项目，最大的陷阱是“从第一个文件开始逐行读”。正确做法是**按依赖的反方向，从骨架到血肉**地扫描，每一步职位回答一个具体问题。

## 七步扫描法（按顺序）

1. **入口与启动路径** —— 项目怎么被用起来？
  找 `__init__.py` 的 `__all__`，`README` 的Quick start，`pyproject.toml` 的 `[project.scripts]`, `api/*server*.py`。
  产出：一句话说清“用户写哪几行代码就能跑起来”。

2. **分层与依赖方向** —— 谁依赖谁？
  看顶层目录划分，`import`的方向。健康的项目依赖是单向的（具体依赖抽象，不反过来）。
  产出：一张分层图（上层 → 下层）。

3. **核心抽象** —— 有哪些“接口/基类”？
  搜`class.*\(ABC\)`, `@abstractmethod`, `Protocol`, `@dataclass`,抽象基类就是项目的“骨架契约”。
  产出：列出所有抽象基类及其“必须实现的方法”。

4. **数据模型** —— 系统里流动的是什么数据？
  搜`@dataclass`, `TypedDict`, `Enum`, `pydantic.BaseModel`，固定的命名空间常量。
  产出：列出核心数据结构和它们的字段。

5. **关键流程（写路径）** —— 数据是怎么进来的？
  找“插入，摄取， index, ingest, pipeline”相关的入口函数，顺着调用链画一遍。
  产出：写路径的阶段序列。

6. **关键流程（读路径）** —— 一次查询经过哪些模块？
  找"query/search/retrieve"入口，顺着画。
  产出：读路径的阶段序列。

7. **可插拔点（扩展机制）** —— 哪些东西是“可替换后端”？
  找`factory`, `registry`, `get_*_class`, `STORAGES = {...}` 这类映射。
  产出：列出所有“一个抽象 → 多个实现”的插拔点。

### 给 Claude Code 的辅助话术（阶段 0 也能用AI）

读码阶段就可以让 AI 当“领读员”，但要**让它输出结构，而不是替你下结论**：

```
我在快速理解 <项目>。请你只做一件事：扫描<目录>，列出其中所有的抽象基类（继承 ABC 或带 @abstractmethod 的），每个类给出：类名，文件位置，它声明的抽象方法签名。不要解释，不要评价，就给一张表。
```


```
顺着 <入口函数> 画出调用链，只到 3 层深。每一层给：函数名 → 它调用的下游函数名。
遇到分支（if/不同 mode）就分叉列出。输出成缩进树，不要散文。
```

> 纪律：阶段 0 用 AI 是为了**更快看清结构**，不是让 AI 替你写设计。设计师阶段 1 junior 自己的产出。
