# 起点：能抄不能造（fluency 有，transfer 无）

用户自述：transform 和 GSAP 都"浅碰过"，照着别人代码写过 demo，知道 GSAP 的 `scroll / to / from`，
但**不懂原理**；给一个新课题让他自己想"该用什么"，"完全不知道，只能照抄"。

含义（决定教什么）：
- 不是语法问题，是**心智模型 + 迁移**问题。他有抄 demo 带来的流利度幻觉，缺长期 storage strength 和组合能力。
- 因此教学不能停在"讲解 API"，每课必须落到一个**可迁移的决策**（给效果 → 调哪个旋钮/曲线），并用 retrieval practice 检验。
- 第一课应从最底层补起：**网页里一张卡片"如何被表示成 3D"**（perspective + preserve-3d + 三轴 + z）——这正是他明确说不懂的点。
- 贯穿全程的框架：**每个 variation = 旋钮 × 映射曲线**。见 [[MISSION.md]]。
