# Agent 可观测性体系（Agent Observability）

> 一句话定义：把传统可观测性的三支柱（Trace / Log / Metric）套用到 Agent 系统上，再补上 AI 特有的两件事——评测（Evaluation）和成本（Cost）——并用一条治理主线（Governance）贯穿始终。

---

# 第一部分：重构后的分层架构（事实层，可以被复用）

## §0 心智模型（先记这个）

数据是**自下而上**流动的，但需求是**自上而下**提出的：

```
                              [人提需求 / 看结果]  ← 自上而下：我想知道 Agent 跑得好不好
   ┌─────────────────────────────────────────────┐
   │  L5 可视化与消费层   (给人看 / 给人用)          │ ↑
   │  L4 评测引擎         (把原始数据变成"好坏"判断)  │ ↑ 数据
   │  L3 数据存储与处理   (存下来 + 加工)            │ ↑ 自
   │  L2 遥测采集层       (从运行中"抽"出数据)        │ ↑ 下
   │  L1 Agent 运行层     (被观测的对象本身)         │ ↑ 而
   │  L0 基础设施层       (一切跑在哪)               │ ↑ 上
   └─────────────────────────────────────────────┘
        ‖  治理与管理 (Governance) —— 横切，贯穿 L0~L5
```
